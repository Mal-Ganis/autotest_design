目标应用 —— Web 登录系统（Login System）
技术建议：纯前端模拟后端
核心原则：功能完整但代码量小，必须有足够的输入验证点、状态转换、业务规则供测试工具分析。
| 模块           | 任务项                            | 实现内容                                                                                                              | 验收标准（AC）                                                                                                                    |
| ------------ | ------------------------------ | ----------------------------------------------------------------------------------------------------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| **1. 基础架构**  | 1.1 项目搭建                       | 搭建 Web 服务；设计数据库表（users: id, username, password\_hash, email, failed\_attempts, locked\_until, created\_at）；配置基础日志 | AC1: 服务能在本地 5000 端口启动；AC2: 数据库能自动初始化；AC3: 提供 README 说明启动命令                                                                  |
|              | 1.2 统一响应格式                     | 定义 API 响应结构：`{code, message, data}`                                                                               | AC1: 所有接口返回统一 JSON 格式；AC2: 异常不抛 500 裸堆栈，返回友好错误码                                                                             |
| **2. 用户注册**  | 2.1 注册接口 `POST /register`      | 接收 username, password, confirm\_password, email；校验规则如下                                                            | AC1: 用户名 3-20 位，仅字母/数字/下划线；AC2: 密码 8-30 位，必须含大写、小写、数字；AC3: 两次密码一致；AC4: 用户名唯一；AC5: 注册成功返回用户ID，密码以 bcrypt 存储                  |
|              | 2.2 注册前端页                      | 表单输入 + 实时格式提示 + 提交按钮                                                                                              | AC1: 能调用后端注册接口；AC2: 前端对空值、长度、密码强度做基础拦截                                                                                      |
| **3. 用户登录**  | 3.1 登录接口 `POST /login`         | 接收 username, password；验证凭据；管理登录状态（Session/JWT）                                                                    | AC1: 正确凭据返回登录成功 + Token/Session；AC2: 错误凭据返回"用户名或密码错误"；AC3: 密码比对使用 bcrypt；AC4: 响应时间 < 500ms（本地）                              |
|              | 3.2 登录状态管理                     | 登录成功后记录 session；登出时清除                                                                                             | AC1: `GET /profile` 需携带有效 Token 才能访问；AC2: `POST /logout` 清除会话；AC3: Token 过期后访问受保护接口返回 401                                   |
|              | 3.3 登录前端页                      | 登录表单 + 错误提示 + 跳转首页                                                                                                | AC1: 登录成功跳转到用户首页；AC2: 登录失败显示具体错误（格式错误/账户锁定/密码错误）                                                                            |
| **4. 账户安全**  | 4.1 登录失败锁定                     | 连续失败 5 次则账户锁定 30 分钟                                                                                               | AC1: 每次失败 `failed_attempts +1`；AC2: 第 5 次失败设置 `locked_until = now + 30min`；AC3: 锁定期间登录返回"账户已锁定，请30分钟后重试"；AC4: 登录成功重置失败计数为 0 |
|              | 4.2 SQL 注入防护                   | 使用 ORM/参数化查询，禁止拼接 SQL                                                                                             | AC1: 使用 sqlmap 或手动输入 `' OR '1'='1` 无法绕过登录                                                                                   |
|              | 4.3 密码安全                       | 禁止明文存储；禁止返回密码字段                                                                                                   | AC1: 数据库中 password 字段为 bcrypt 哈希；AC2: 任何接口响应不包含 password 字段                                                                 |
| **5. 密码重置**  | 5.1 重置请求 `POST /reset-request` | 接收 email，生成重置令牌（简化版可直接返回链接或模拟发送）                                                                                  | AC1: 邮箱存在则生成 token；AC2: 邮箱不存在也返回相同提示（防枚举）                                                                                   |
|              | 5.2 重置确认 `POST /reset`         | 接收 token, new\_password, confirm\_password                                                                        | AC1: Token 有效且未过期；AC2: 新密码符合注册时的复杂度规则；AC3: 重置成功后旧 Token 失效                                                                  |
| **6. 非功能需求** | 6.1 性能                         | 接口响应优化                                                                                                            | AC1: 登录/注册接口 P95 响应时间 < 2 秒（本地测试）；AC2: 支持至少 50 并发登录请求不崩溃（可用 Locust 压测）                                                      |
|              | 6.2 可用性                        | 错误提示友好、界面简洁                                                                                                       | AC1: 所有表单错误有明确中文提示；AC2: 页面在手机浏览器可正常显示                                                                                       |
|              | 6.3 可维护性                       | 代码分层（Controller/Service/DAO）                                                                                      | AC1: 业务逻辑与数据访问分离；AC2: 核心函数有 docstring 注释                                                                                    |
