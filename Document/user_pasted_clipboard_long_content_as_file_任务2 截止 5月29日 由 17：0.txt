描述

FR 1.0

输入/解析

系统可以从多种来源（例如CSV、纯文本或直接用户输入）导入软件需求。

FR 1.1

需求结构

系统可以解析和分词原始文本，识别输入字段、数据范围、条件和预期动作等关键组成部分。

FR 2.0

风险分析与优先级

系统可以为每个导入需求分配风险评分和测试优先级（高、中、低）。

FR 3.0

黑盒测试设计

该系统能够自动应用并生成至少三种来自ISO 29119-4核心黑箱技术的测试用例（例如，等效划分、边界值分析、决策表）。

FR 4.0

白盒测试建模

该系统可以建模系统行为（例如，状态转换图），并为所选覆盖标准（例如，所有状态）生成最优测试序列。

FR 5.0

测试神谕生成

该系统可以帮助综合特定需求和特定测试数据的预期结果。

FR 6.0

输出与出口

该系统可以生成结构化、标准格式（如 JSON 或 Excel/CSV）的测试产物（测试用例、测试套件、风险评分），并适合导入测试管理工具。

FR 7.0

测试套件优化

该系统包含基于风险或覆盖效率对生成测试套件进行优先级排序或最小化的优化。

该任务要求开发一款由人工智能驱动的AutoTestDesign工具，能够执行需求分析、风险评估和系统化测试用例生成等功能，并符合行业标准，如ISTQB基础级原则及ISO/IEC/IEEE标准中定义的详细测试技术。

重要澄清：你将开发AutoTestDesign工具，然后用它测试一个自选的目标应用（例如网页应用、计算器、登录模块、待办事项列表），以展示该工具的有效性。

1.1 AutoTestDesign 要求

AutoTestDesign 应用程序包含功能需求（1.1.1）和非功能需求（1.1.2）。

1.1.1 功能需求（FR）

系统必须至少实现FR 1.0、FR 1.1、FR 2.0、FR 3.0、FR 6.0，以及“主要”段落中描述的交互式审查功能。其他FR（4.0、5.0、7.0）是可选的，但实际应用会获得额外学分。
